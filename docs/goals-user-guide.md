# Руководство пользователя: сервис «Целеполагание» (РИТМ)

Документ подготовлен на основе тестовых сценариев и описывает типовые операции в модуле целеполагания системы **РИТМ**.

> **PDF по разделам:** готовые файлы в папке [`goals-user-guide/pdf/`](goals-user-guide/pdf/). Пересборка: `node scripts/build-goals-pdf.mjs` (нужен пакет `md-to-pdf`).

---

## 1. Назначение сервиса

Сервис **«Целеполагание»** предназначен для:

- постановки и согласования **карт результативности (КР)** сотрудников;
- ведения **ключевых показателей эффективности (КПЭ)** в **реестре КПЭ**;
- контроля плановых и фактических значений в **таблице ПФК** (плановые, фактические и критические значения);
- **делегирования** полномочий по ролям;
- **оценки результативности** по итогам периода.

**Путь в системе:** боковое меню → **Целеполагание** (`/services/goals`).

---

## 2. Структура раздела

На странице «Целеполагание» доступны четыре вкладки:

| Вкладка | Назначение |
|--------|------------|
| **Карты результативности** | Список КР, создание, редактирование, согласование, оценка |
| **Реестр КПЭ** | Справочник показателей, используемых в КР и ПФК |
| **Таблица ПФК** | План, база/критическое отклонение, факт, статусы плана и факта |
| **Делегирование** | Передача ролей и полномочий другим сотрудникам |

**Личный кабинет** (профиль пользователя) дополняет раздел: в блоке вашей роли отображаются **ожидаемые действия** (например, «Необходимо акцептовать КР», «Необходимо внести плановые значения КПЭ»). Клик по такому действию ведёт к нужной вкладке или задаче.

Для создания КР из контекста команды в сценариях также используется вкладка **«Моя команда»** в личном кабинете.

---

## 3. Основные понятия

| Термин | Расшифровка |
|--------|-------------|
| **КР** | Карта результативности — документ целей сотрудника за период |
| **КПЭ** | Ключевой показатель эффективности |
| **ЗРД** | Задание по развитию |
| **ККПЭ** | Корректирующий КПЭ |
| **ПФК** | Таблица плановых, фактических и критических значений |
| **ГЦ** | Группа целеполагания |
| **ЦОР** | Цикл оценки результативности (период) |
| **ССП / ВСП** | Структурные подразделения в иерархии |

### 3.1. Состав карты результативности

В КР указываются:

1. **КПЭ** — вес (5–75%), вид (основной / мониторинговый), зона ответственности;
2. **ЗРД** — наименование, вес (5–75%), ожидаемый результат;
3. **ККПЭ** — при необходимости; вид «Корректирующий», зона ответственности.

**Правило:** сумма весов всех **КПЭ** и **ЗРД** в карте должна быть **100%**.

### 3.2. Статусы карты результативности

| Статус | Смысл |
|--------|--------|
| **Черновик** | Карта в работе у руководителя, не отправлена на согласование |
| **Разработка** | Карта на акцептовании (согласовании) |
| **Разработка (!)** | Разработка с отклонением / требует внимания |
| **Активная** | Карта утверждена, используется в периоде |
| **Архивная** | Завершённый период (для просмотра и переиспользования) |

### 3.3. Статусы плана и факта в ПФК

**План:**

| Статус | Кто действует дальше |
|--------|----------------------|
| Ожидание ввода плана | Аналитик ПЛАН |
| Ожидание утверждения плана | Руководитель ГЦ |
| План утверждён | Далее — ввод факта |
| План отклонён | Аналитик ПЛАН (доработка) |

**Факт:**

| Статус | Кто действует дальше |
|--------|----------------------|
| Ожидание ввода факта | Аналитик ФАКТ |
| Ожидание утверждения факта | Верификатор |
| Факт утверждён | Расчёт выполнения КПЭ |
| Факт отклонён | Аналитик ФАКТ (доработка) |

---

## 4. Роли и зоны ответственности

| Роль | Основные задачи в сервисе |
|------|---------------------------|
| **Руководитель** | Создание и ведение КР подчинённых, отправка на акцептование, оценка результативности |
| **Координатор** | Акцептование КР в статусе «Разработка» по своей ГЦ |
| **Методолог ДОМРР / ДРП** | Экспертиза **верхнеуровневых** КР (двойное визирование) |
| **Руководитель ГЦ** | Утверждение / отклонение плановых значений в ПФК |
| **Аналитик ПЛАН** | Ввод и корректировка плана, импорт из Excel |
| **Аналитик ФАКТ** | Ввод и корректировка факта, импорт из Excel |
| **Верификатор** | Утверждение / отклонение фактических значений |

Роли назначаются в платформе **Бейз Коммандер**.

## 5. Руководитель: работа с картами результативности

### 5.1. Создание КР для сотрудника

**Предусловия:** вы вошли в РИТМ; роль — «Руководитель».

1. Откройте **Целеполагание** → вкладку **«Карты результативности»**  
   *Альтернатива из сценария:* личный кабинет → **«Моя команда»** → таблица сотрудников.
2. Найдите сотрудника. В столбце **«Карты результативности, периоды»** нажмите **бейдж** нужного периода (например, «2026 год», «1Q2026»).
3. В поповере нажмите **«Создать КР»**.
4. Заполните карту:
   - **Добавить КПЭ** → выберите показатели из списка (доступны КПЭ из ГЦ сотрудника и кроссфункциональные);
     - **«Доступно»** — период КПЭ совпадает с периодом карты;
     - **«Недоступно»** — период не совпадает (строка заблокирована);
     - для деталей — значок **«Глаз»**;
     - отметьте чекбоксы → **«Выбрать КПЭ»** → **«Сохранить»**;
     - в карте укажите **вес**, **вид**, **зону ответственности**;
   - **Добавить ЗРД** → наименование, вес, ожидаемый результат (можно несколько);
   - **Добавить ККПЭ** — аналогично КПЭ; вид «Корректирующий» проставится автоматически.
5. Нажмите **«Сохранить»** внизу формы.

**Результат:** КР в статусе **«Черновик»**, отображается в списке команды.

**Важно:**

- сумма весов КПЭ и ЗРД = **100%**;
- удаление элемента — **«✕»** рядом с строкой;
- при закрытии без сохранения система предложит сохранить изменения.

---

### 5.2. Переиспользование КР (один сотрудник или группа)

1. **Целеполагание** → **«Карты результативности»**.
2. Найдите карту-источник (любой статус). Нажмите бейдж периода → **«Открыть КР»**.
3. Меню **«…»** (три точки) → **«Переиспользовать карту результативности»**.
4. На странице **«Создание групп карт результативности»** нажмите **«Выбрать сотрудников»**.
   - **«Доступно»** — можно создать КР на период;
   - **«Не доступно»** — у сотрудника уже есть цели на этот период.
5. **«Сохранить»** → проверьте период → **«Создать карты результативности»** → в диалоге **«Создать»**.
6. Уведомление: *«Карты результативности успешно созданы!»* → **«Закрыть»**.

**Результат:** копии в статусе **«Черновик»** для выбранных сотрудников.

---

### 5.3. Просмотр КР подчинённых

1. **Целеполагание** → **«Карты результативности»**.
2. **Фильтр «Период»** — выберите год.
3. Чекбокс **«Сотрудники прямого подчинения»**:
   - включён — только прямые подчинённые;
   - выключен — все сотрудники подразделения.
4. Используйте поиск по **ФИО** и фильтры в заголовках: **Группа целеполагания**, **Должность**, **Подразделение**.
5. Клик по **бейджу** периода → поповер: период, статус, **«Открыть КР»** (для статусов: Черновик, Разработка, Активная, Архивная).

---

### 5.4. Редактирование КР в статусе «Черновик»

1. Откройте КР (бейдж → **«Открыть КР»**), убедитесь в статусе **«Черновик»**.
2. Измените: тип/период, срок действия, КПЭ, ЗРД, ККПЭ.
3. **«Сохранить»** → сообщение *«Карта результативности успешно изменена»*.

Карта остаётся в **«Черновик»**. Для запуска согласования — отдельно **«Отправить на акцептование»**.

Меню **«…»**: карта руководителя сотрудника, история изменений, переиспользование.

---

### 5.5. Отправка КР на акцептование (Черновик → Разработка)

1. Откройте КР в **«Черновик»**, проверьте заполнение и **сумму весов = 100%**.
2. **«Отправить на акцептование»**.
3. Сообщение *«Статус успешно изменен»*; статус → **«Разработка»**; вместо отправки появится **«Редактировать КР»**.

**Дальше:** карта доступна координатору (и методологам — для верхнеуровневых КР).

---

### 5.6. Редактирование КР в статусах «Разработка» / «Активная»

1. Откройте КР → **«Редактировать КР»**.
2. Подтвердите в диалоге: *«…перевести КР из статуса "Разработка" в статус "Черновик"? …потребуется повторный запуск процедуры акцептования»* → **«Да»**.
3. Статус → **«Черновик»**, поля доступны для правок.
4. **«Сохранить»** → при необходимости снова **«Отправить на акцептование»**.

**Важно:** прежнее согласование аннулируется; история фиксируется в **«…» → «Посмотреть историю карты результативности»**.

---

### 5.7. Утверждение оценки результативности

**Предусловия:** КР **«Активная»**; по КПЭ в ПФК план утверждён, факт верифицирован, выполнение рассчитано.

1. **Целеполагание** → **«Карты результативности»** → откройте активную КР.
2. Заполните:
   - **«Выполнение (оценка руководителя)»** — текстовая обратная связь;
   - **«Оценка руководителя, %»** — итоговый процент.
3. **«Сохранить»** → появится **«Предварительная оценка результативности»** и активируется **«Утвердить оценку»**.
4. **«Утвердить оценку»** → *«Оценка утверждена!»*; поля блокируются, процесс завершён.

---

## 6. Координатор: акцептование КР

**Предусловия:** роль «Координатор»; есть КР в **«Разработка»** по вашей ГЦ.

1. **Целеполагание** → в личном кабинете: **«Необходимо акцептовать КР»** или вкладка **«Карты результативности»**.
2. Отфильтруйте статус **«Разработка»**.
3. Откройте карту — клик по **ФИО** в таблице.
4. Проверьте КПЭ, ЗРД, ККПЭ, веса, сроки.

**Отклонение:**

- **«Отклонить»** → укажите причину (обязательно) → **«Отправить»**.
- Карта: **«Разработка (отклоненная)»**, возврат руководителю.

**Акцептование:**

- **«Акцептовать»**:
  - **каскадируемая КР** → статус **«Активная»** (согласование завершено);
  - **верхнеуровневая КР** → остаётся **«Разработка»**, далее — методологи ДОМРР/ДРП.

---

## 7. Методолог ДОМРР / ДРП: верхнеуровневые КР

**Предусловия:** верхнеуровневая КР после акцепта координатора, статус **«Разработка»**.

1. Личный кабинет → **«Необходимо акцептовать КР»** или **«Карты результативности»**.
2. Откройте карту по **ФИО**, проведите методологическую экспертизу.

**Отклонение** — как у координатора; карта → **«Разработка (отклоненная)»**.

**Акцептование:**

- Если второй методолог **ещё не** акцептовал — карта остаётся в **«Разработка»**, ваш этап зафиксирован.
- Если **оба** методолога (ДОМРР и ДРП) акцептовали — статус **«Активная»**.

Используйте **«…»** → история карты, карта руководителя для проверки каскада.

---

## 8. Таблица ПФК: общие элементы интерфейса

**Переход:** **Целеполагание** → вкладка **«Таблица ПФК»** (или кнопка **«ПФК»** / ожидаемое действие в личном кабинете).

| Элемент | Назначение |
|---------|------------|
| **Полный вид** | Дополнительные столбцы: формула, аналитики, руководитель ГЦ, верификатор |
| **Режим редактирования** | Ввод/утверждение значений; кнопки **«Сохранить»** и **«Закрыть»** |
| **Экспорт/Импорт** | Массовая выгрузка и загрузка Excel |
| **Действие** (в строке) | История операций по КПЭ |
| Фильтры в заголовках | Быстрый отбор по статусам, периоду, зоне ответственности |

Типовые столбцы: ID, наименование КПЭ, ед. изм., период, зона ответственности, **План**, **База/Крит. откл.**, **Статус плана**, **Факт**, **Статус факта**, % выполнения.

---

## 9. Аналитик ПЛАН

### 9.1. Ввод плановых значений

1. Личный кабинет → **«Необходимо внести плановые значения КПЭ»** или **ПФК**.
2. При необходимости включите **«Полный вид»** и **«Режим редактирования»**.
3. Отфильтруйте **«Ожидание ввода плана»**.
4. Введите **План** и **База/Крит. откл.** → **«Сохранить»**.

**Результат:** статус → **«Ожидание утверждения плана»** → далее Руководитель ГЦ.

### 9.2. Редактирование плана

| Исходный статус | После сохранения |
|-----------------|------------------|
| Ожидание утверждения плана | Значение обновлено, статус без изменений |
| План утверждён | Диалог *«Вы изменяете утверждённое значение…»* → при **«Продолжить»** статус снова **«Ожидание утверждения плана»** |

### 9.3. Импорт из Excel

1. **Экспорт/Импорт** → **«Экспорт Таблица ПФК краткий вид»** → файл `Таблица ПФК краткая форма.xlsx`.
2. Заполните только столбцы **«План»** и **«База/ Крит.откл»** (не меняйте структуру и ID).
3. **Импорт Таблица ПФК краткий вид** → выберите файл → *«Успешно. Таблица обновилась!»*.

**Совет:** перед экспортом примените фильтры — в файле будут только нужные строки.

---

## 10. Руководитель ГЦ: утверждение плана

1. **«Необходимо утвердить плановые значения КПЭ»** или вкладка **ПФК**.
2. Фильтр: **«Ожидание утверждения плана»**.
3. **«Режим редактирования»** → в строке:
   - **«Утвердить план»** (✅) → **«План утверждён»**;
   - **«Отклонить план»** (❌) → комментарий обязателен → **«План отклонён»** (доступно **«Отменить отклонение»**).
4. **«Сохранить»** — зафиксировать все решения.

---

## 11. Аналитик ФАКТ

### 11.1. Ввод фактических значений

1. **«Необходимо внести фактические значения КПЭ»** или **ПФК**.
2. **«Полный вид»**, **«Режим редактирования»**, фильтр **«Ожидание ввода факта»**.
3. Заполните столбец **«Факт»** → **«Сохранить»**.

**Результат:** **«Ожидание утверждения факта»** → Верификатор.

### 11.2. Редактирование и импорт факта

Логика как у плана (см. п. 9.2–9.3), но для столбца **«Факт»** и статусов факта.

---

## 12. Верификатор: утверждение факта

1. **«Необходимо верифицировать фактические значения КПЭ»** или **ПФК**.
2. Фильтр **«Ожидание утверждения факта»**, **«Режим редактирования»**.
3. **«Утвердить факт»** (✅) или **«Отклонить факт»** (❌) с обязательным комментарием.
4. **«Сохранить»**.

**После утверждения:** расчёт **значения выполнения КПЭ**; сбор данных по показателю завершён.

---

## 13. Делегирование полномочий

Вкладка **«Делегирование»**:

| Подвкладка | Содержание |
|------------|------------|
| **Мои роли** | Ваши роли и действия; роли, делегированные вам другими |
| **Управление делегированием** | Назначение роли сотруднику на период, история |

При делегировании передаётся **весь функционал роли**. Укажите период **с** / **по** (пустое «по» — до текущего момента). Завершение — **«Завершить»** в истории.

---

## 14. Реестр КПЭ

Вкладка **«Реестр КПЭ»** — единый каталог показателей для КР и ПФК.

- просмотр и поиск КПЭ;
- **«+ Создать»** — новый показатель (при наличии прав);
- **«Полный вид»**, **«Экспорт»**, фильтры по столбцам.

Изменения в реестре отражаются в связанных записях ПФК.

---

## 15. Сквозной жизненный цикл (краткая схема)

```
КР: Черновик → [Отправить] → Разработка → [Координатор / Методологи] → Активная
         ↑___________________________[Редактировать КР]________________________|

ПФК: Ожидание ввода плана → [Аналитик ПЛАН] → Ожидание утверждения плана
     → [Руководитель ГЦ] → План утверждён
     → Ожидание ввода факта → [Аналитик ФАКТ] → Ожидание утверждения факта
     → [Верификатор] → Факт утверждён → расчёт выполнения

КР (Активная): оценка руководителя → утверждение оценки результативности
```

## 16. Частые ошибки и подсказки

| Ситуация | Что делать |
|----------|------------|
| Нельзя выбрать КПЭ при создании КР | Проверьте период карты и период оценки КПЭ |
| Не отправляется на акцептование | Сумма весов КПЭ + ЗРД должна быть 100% |
| «Не доступно» при переиспользовании | У сотрудника уже есть КР на период — редактируйте существующую |
| Импорт Excel не проходит | Не переименовывайте и не удаляйте столбцы; меняйте только данные |
| Изменили утверждённый план/факт | Статус сбросится на повторное утверждение — предупреждение системы |
| Нет кнопки «Сохранить» в КР | Карта не в «Черновик» — используйте «Редактировать КР» |
